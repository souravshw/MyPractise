Understanding Assign Cookies in Java

Learn how to maximize content children using cookies of different sizes.

Real-World Analogy: The Cookie Handout

You have hungry children (with greed values) and cookies of different sizes. To satisfy the maximum children, you sort both children and cookies. You try to satisfy the child with the smallest greed using the smallest possible cookie that satisfies them.

Step-by-Step Execution Walkthrough

Sort greed array g and cookie size array s. Use two pointers 'child' and 'cookie'. If s[cookie] >= g[child], increment child. Increment cookie in all cases. Return child index.

Java Implementation Code

Below is the complete, self-contained Java source code that solves this problem. It also includes a main method that traces the execution with console outputs.

package io.practise.dsa;

import java.util.Arrays;

public class AssignCookies {

    // Sort + Two Pointers - O(n log n)
    public int findContentChildren(int[] g, int[] s) {
        Arrays.sort(g);
        Arrays.sort(s);
        int child = 0, cookie = 0;
        while (child < g.length && cookie < s.length) {
            if (s[cookie] >= g[child]) {
                child++;
            }
            cookie++;
        }
        return child;
    }

    public static void main(String[] args) {
        AssignCookies solver = new AssignCookies();
        int[] g = {1, 2, 3}; // children greed
        int[] s = {1, 1};    // cookies size

        System.out.println("--- Assign Cookies Demonstration ---");
        System.out.println("Children greed: " + Arrays.toString(g));
        System.out.println("Cookies size: " + Arrays.toString(s));
        System.out.println("Content children: " + solver.findContentChildren(g, s));
    }
}

Conclusion

Solving the Assign Cookies problem highlights how we can leverage key data structures and algorithmic paradigms (like two pointers, hash maps, or dynamic programming) to significantly optimize runtime and simplify code complexity.
