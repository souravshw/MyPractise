Under the Hood: Building a Custom HashMap in Java

Honestly, Java's HashMap is one of those classes we all import and use daily without thinking twice. We call put(), we call get(), and we just expect $O(1)$ lookups. But have you ever actually stopped to look under the hood to see how it organizes memory under the hood?

I decided to write a custom, bare-bones HashMap implementation from scratch to show exactly how hashing, collision chains, and dynamic resizing function behind the scenes. Before diving into the Java code, let's look at how the core concept works in the real world.

Think of it like this:

Imagine a post office with only 16 mailboxes (these are our array Buckets). When a letter arrives, the clerk needs to decide which mailbox it goes into based on the recipient's name (the Key):

  • Hashing (Choosing the Box): The clerk runs the name through a quick rule—like taking the name's length—and gets a box number between 0 and 15. The letter always goes there.
  • Collisions (Sharing Boxes): Since there are thousands of possible names and only 16 mailboxes, people will inevitably share a box. If both "Alice" and "Alex" hash to box 5, the clerk chains the letters together inside box 5 using paperclips. This is Chaining.
  • Rehashing (Expanding the Boxes): As more letters arrive, the paperclipped chains in each mailbox get longer, making it slow for the clerk to find a letter. To fix this, the post office doubles the mailboxes to 32, recalculates the box number for every single name, and redistributes them.

Core Technical Concepts

1. The Bucket Array and Nodes

In our implementation, we store elements in a simple array of Nodes. If multiple keys hash to the same bucket index, we link them together using a singly linked list. Each Node holds the key, the value, and a reference to the next node in the chain.

static class Node<K, V> {
    final K key;
    V value;
    Node<K, V> next;
    // Constructor...
}

2. Calculating the Bucket Index (Hash Function)

When inserting a key, we get its hash code (Java's built-in integer identifier for objects) and fit it within our array capacity using a modulo operation. We explicitly assign null keys to index 0:

private int hash(K key) {
    if (key == null) return 0;
    return Math.abs(key.hashCode() % capacity);
}

3. Putting Elements (Inserting/Updating)

When saving a key-value pair, we hash the key to find the index. If the bucket is empty, we place our new node there. If the bucket already has a node (a collision), we walk down the chain:

  • If we find a node with the same key (comparing with equals()), we overwrite its value.
  • If we reach the end of the chain without finding a duplicate, we append our new node.

4. Getting Elements (Retrieval)

To retrieve a value, we hash the key, jump straight to the correct bucket index, and search down the linked list comparing keys. If we find a match, we return its value; if not, we return null.

5. Resizing and Rehashing

If we have too many elements compared to the size of the array, collisions will spike and slow down lookups. We use a threshold called the Load Factor (usually 0.75). Once the count of elements exceeds 75% of our array's size, we double the capacity, allocate a new array, and rehash all keys to distribute them evenly across the new indexes.

Complete Source Code

Here is the full CustomHashMap.java implementation containing these concepts with verbose console logs to show the steps in action.

package io.practise.map;

public class CustomHashMap<K, V> {
    
    static class Node<K, V> {
        final K key;
        V value;
        Node<K, V> next;

        public Node(K key, V value, Node<K, V> next) {
            this.key = key;
            this.value = value;
            this.next = next;
        }
    }

    private Node<K, V>[] table;
    private int capacity;
    private final float loadFactor = 0.75f;
    private int size;

    @SuppressWarnings("unchecked")
    public CustomHashMap(int initialCapacity) {
        this.capacity = initialCapacity;
        this.table = (Node<K, V>[]) new Node[capacity];
        this.size = 0;
    }

    private int hash(K key) {
        if (key == null) return 0;
        return Math.abs(key.hashCode() % capacity);
    }

    public V put(K key, V value) {
        if ((float) (size + 1) / capacity > loadFactor) {
            resize();
        }

        int index = hash(key);
        Node<K, V> head = table[index];

        if (head == null) {
            table[index] = new Node<>(key, value, null);
            size++;
            return null;
        }

        Node<K, V> curr = head;
        while (curr != null) {
            if (isKeyEqual(curr.key, key)) {
                V oldValue = curr.value;
                curr.value = value;
                return oldValue;
            }
            if (curr.next == null) break;
            curr = curr.next;
        }

        curr.next = new Node<>(key, value, null);
        size++;
        return null;
    }

    public V get(K key) {
        int index = hash(key);
        Node<K, V> curr = table[index];

        while (curr != null) {
            if (isKeyEqual(curr.key, key)) return curr.value;
            curr = curr.next;
        }
        return null;
    }

    public V remove(K key) {
        int index = hash(key);
        Node<K, V> curr = table[index];
        Node<K, V> prev = null;

        while (curr != null) {
            if (isKeyEqual(curr.key, key)) {
                V value = curr.value;
                if (prev == null) {
                    table[index] = curr.next;
                } else {
                    prev.next = curr.next;
                }
                size--;
                return value;
            }
            prev = curr;
            curr = curr.next;
        }
        return null;
    }

    @SuppressWarnings("unchecked")
    private void resize() {
        int oldCapacity = capacity;
        Node<K, V>[] oldTable = table;

        capacity = oldCapacity * 2;
        table = (Node<K, V>[]) new Node[capacity];
        size = 0;

        for (int i = 0; i < oldCapacity; i++) {
            Node<K, V> curr = oldTable[i];
            while (curr != null) {
                put(curr.key, curr.value);
                curr = curr.next;
            }
        }
    }

    private boolean isKeyEqual(K k1, K k2) {
        if (k1 == null) return k2 == null;
        return k1.equals(k2);
    }
}

Summary

By implementing a custom map, we see how arrays and linked lists combine to form one of Java's most powerful data structures. The performance of lookups is kept close to constant time O(1) by using hash codes to index directly into the bucket array, resolving occasional collisions dynamically using chained lists.
