Understanding Design Hit Counter in Java

Learn how to design a hit counter that records hits in the past 5 minutes.

Real-World Analogy: The Sliding Turnstile

Imagine counting people entering a museum in the last 5 minutes. You store timestamps of visitors in a queue. When you request the hit count, you look at the front of the queue and remove all timestamps older than 5 minutes (300 seconds). The size of the remaining queue is your count.

Step-by-Step Execution Walkthrough

hit(timestamp) offers timestamp to queue. getHits(timestamp) polls elements from queue as long as (timestamp - peek) >= 300. Returns queue size.

Java Implementation Code

Below is the complete, self-contained Java source code that solves this problem. It also includes a main method that traces the execution with console outputs.

package io.practise.dsa;

import java.util.*;

public class DesignHitCounter {

    public static class HitCounter {
        private Queue<Integer> queue = new LinkedList<>();

        public void hit(int timestamp) {
            queue.offer(timestamp);
        }

        public int getHits(int timestamp) {
            while (!queue.isEmpty() && timestamp - queue.peek() >= 300) {
                queue.poll();
            }
            return queue.size();
        }
    }

    public static void main(String[] args) {
        HitCounter counter = new HitCounter();
        System.out.println("--- Design Hit Counter Demonstration ---");
        counter.hit(1);
        counter.hit(2);
        counter.hit(3);
        System.out.println("Hits at timestamp 4: " + counter.getHits(4)); // 3
        System.out.println("Hits at timestamp 300: " + counter.getHits(300)); // 3
        System.out.println("Hits at timestamp 301: " + counter.getHits(301)); // 2 (hit at 1 expired)
    }
}

Conclusion

Solving the Design Hit Counter problem highlights how we can leverage key data structures and algorithmic paradigms (like two pointers, hash maps, or dynamic programming) to significantly optimize runtime and simplify code complexity.
